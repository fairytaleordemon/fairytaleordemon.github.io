Spring-Boot + Oracle集成问题

Posted on 2016-06-30

背景

以前都是使用Spring + Hibernate + Mysql 来搭建的开发环境，今天需要使用到Oracle。搭建过程中遇到的问题和大家分享下。

本地环境

环境名称 版本 备注
Mac OS
Docker Oracle 安装在docker 下的
Oracle 11g
Maven 3.3.5 下载jar包
Spring Tools Suite 类似于Eclipse

Maven 无法下载Oracle的包

问题描述

导入以下jar包，在mvnrepository.com可以查到关于 oracle 依赖的jar包，但是就是下载不出来。引入的包如下。 

<dependency>  
    <groupId>com.oracle</groupId>  
    <artifactId>ojdbc6</artifactId>   
    <version>11.2.0.2.0</version>   
</dependency>

但是包一直下载不下来，查阅了相关资料之后，发现oracle的驱动包都收费了。无法从maven仓库中直接下载。

解决方案

在 Oracle 官网上下载 ojdbc6.jar ，如果有这个 jar包就请忽略这步。

在 Console 中运行以下命令

mvn install:install-file -DgroupId=com.oracle "-DartifactId=ojdbc6"   
"-Dversion=11.2.0.2.0" "-Dpackaging=jar" "-Dfile=Downloads/  
ojdbc6.jar"

替换 —Dfile为自己ojbc6的下载路径。运行完成后会安装在本地的内仓中。

ORA-12505, TNS:listener does not currently know of SID given in connect descriptor

问题描述

我的连接串如下

spring.datasource.url=jdbc:oracle:thin:@localhost:1521:XE?  
autoReconnect\=true&useUnicode\=true&characterEncoding\=UTF-8

不知道你们看出问题没有，我之前都用的是 Mysql 所以没看出来，找了很多blog。都建议先去查 Oracle 的 sid 到底是多少。命令如下： 

select INSTANCE_NAME from v$instance;

由于我使用的是Docker，所以操作步骤如下，没有使用的直接看最后。

docker exec -ti oracle bash
#进入 oracle 容器，我这里的 oracle 是我容器的名称。
sqlplus 
#就是登陆 oracle 了，执行完让你输入用户名密码
#最后执行以上的查询命令。

操作完成后发现，我的SID确实是XE，没有错啊！一脸懵逼，出去转了一圈，然后回来，突然发现，oracle 的 url 貌似不能像 Mysql一样指定参数，随后把XE后面的东西全部删掉了。那么为什么一开始会加呢，因为偷懒把之前的 Mysql的连接串直接cp 过来了。再启动程序，问题解决，下面附一张连接池的表。

连接池

Mysql 连接串

jdbc:mysql://localhost/dbname?characterEncoding=gb2312  
com.mysql.jdbc.Driver  
org.hibernate.dialect.MySQLDialect

Sql Server 

jdbc:jtds:sqlserver://localhost:1433;DatabaseName=dbname  
net.sourceforge.jtds.jdbc.Driver  
org.hibernate.dialect.SQLServerDialect

Oracle 

jdbc:oracle:thin:@localhost:1521:dbname  
oracle.jdbc.driver.OracleDriver  
org.hibernate.dialect.OracleDialect
